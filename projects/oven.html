Project Description

The purpose of the project is to simulate the operation of an oven by implementing several relevant tasks. Python was used to create the graphical user interface (GUI) to control the operation of the oven and give live temperature feedback. An Arduino microcontroller was used to perform the various hardware tasks. Serial communication was used to send commands from the GUI to the microcontroller. This oven control simulator was completed with one other student as a final project for a Mechatronics course.

Task Overview

The complete operation of the simulated oven can be broken up into 8 separate tasks that were implemented. Most tasks required corresponding functions in both Python and C++. The tasks are as follows:

  • Indicate when the oven is turned on in the GUI
  • Track the temperature inside the oven as a function of time
  • Indicate when the temperature of the oven is hot with a yellow LED
  • Indicate when the oven is fully heated with a green LED
  • Use a red LED to flash periodically when the oven is on
  • Activate a buzzer when the oven is fully heated
  • Indicate the temperature on an analog scale using a servo motor
  • Control the heating rate using a potentiometer

For each of the tasks, a state-transition diagram was used to accurately track the states of each component. The diagram also makes the program logic much easier. Figure 1 shows the state-transition diagram for temperature. The system starts in the “Idle” state, and transitions to the “Off” state when the “start” signal is received. The machine transitions to the “On” state when the “on” signal is received. If the oven is turned back “off”, the oven returns to the “Off” state. In the “On” state, the heat is applied, and the temperature increases. Once the oven is fully heated, the machine moves to the “Steady” state, which maintains the temperature within a close range. If the desired temperature of the oven is increased, the system goes back to the “On” state to continue heating.

Python Implementation

The GUI for the application was created using the “tkinter” Python module. An image of the screen when the program starts is shown in Figure 2. When the “Start” button is pressed, the “On” and “Off” radio buttons, along with the temperature slider, become active. The control mode window becomes active when the oven is turned on. The heating rate is controlled by a potentiometer, which can be set from 1–3 deg/s. This operation was to simulate ovens of different sizes and heating capabilities. The green label indicates when the oven is ready (corresponds to the green LED), and the yellow label is to indicate when the oven is hot (corresponds to the yellow LED). A plot on the right side of the interface displays temperature as a function of time once the oven is active.

Once the oven is activated, an animation of the plot becomes active. The heat increases linearly until the oven temperature is raised just above the desired temperature. The oven constantly turns on and off to maintain the desired temperature within a few degrees. An image of the GUI showing the process over time is given in Figure 3. It is important to note that the temperature range on the slider is chosen arbitrarily low. This decision was to allow the oven to heat up at a reasonable pace. These values do not correspond to typical oven temperatures.

The Python program also has a dynamic mode feature, which more accurately represents the heating process. A first-order heating model can be enabled to get a more realistic simulation. In this mode, the potentiometer controls the inverse of the time constant for the exponential response. An image of the GUI using this dynamic mode is provided in Figure 4.

Arduino Implementation

When an event occurs in the Python program, signals are sent to a serial port for the Arduino to read. The “serial” Python library is used to send the data. The commands are received by the Arduino, and depending on the type of command, certain functions are updated in the program. For most of the functions in the Python code, there is a corresponding function in the microcontroller implementation. To avoid slow operation, entry code was applied to both programs. Entry code is the process of including entry and exit flags in each function to avoid spamming the serial port with redundant commands. This feature allows single commands to be sent to the terminal from Python, and that each hardware component receives only a single command per state change.

The microcontroller was responsible for activating each of the three LEDs, activating the buzzer, turning the servo motor, and receiving analog inputs from the potentiometer. A circuit diagram automatically generated from Tinkercad is shown in Figure 5. The schematic shows additional information, such as the resistors chosen for the LEDs and the pins used on the Arduino.

Testing

Once development was complete, the circuit was wired based on the schematic and tested. An image of the circuit built on a breadboard is shown in Figure 6. The Arduino is powered by a laptop through the USB cable, which supplies power to all components on the board.

A live demonstration of the oven simulator is given in Video 1. The video walks through all of the tasks outlined above. The dynamic mode was enabled, which incorporates the more accurate heating model. The potentiometer was also used to show the impact of heating the oven faster. A screen recording of the GUI is also shown in the video allowing comparison between the interface and hardware outputs in real time.
